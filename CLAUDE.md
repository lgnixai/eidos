## Project Overview

Eidos is an extensible framework for managing personal data throughout your lifetime in one place. It's built as both a web application and desktop application (via Electron) with offline-first capabilities, AI integration, and a powerful extension system.

## Architecture

This is a TypeScript/JavaScript monorepo using pnpm workspaces with the following key structure:

### Core Applications

- **`apps/web-app/`** - Main web application (React + Vite)
- **`apps/desktop/`** - Desktop application (Electron wrapper)
- **`apps/docs/`** - Documentation site (Astro)

### Shared Packages

- **`packages/core/`** - Core data management, SQLite operations, and business logic
- **`packages/ai/`** - AI integration layer with multiple LLM providers
- **`packages/lib/`** - Shared utilities, storage, embedding, and common functions
- **`packages/locales/`** - Internationalization resources
- **`packages/shared/`** - Shared TypeScript types and Vite configuration
- **`packages/worker/`** - Web workers for background processing
- **`packages/v3/`** - V3 compiler and code processing tools

### Technology Stack

- **Frontend**: React 18, TypeScript, Tailwind CSS, Vite
- **UI Components**: Radix UI, Shadcn UI, Lexical (rich text editor)
- **State Management**: Zustand, SWR for data fetching
- **Database**: SQLite with better-sqlite3, custom SQL parsing and processing
- **AI Integration**: OpenAI, Anthropic, Google AI, Mistral AI via @ai-sdk
- **Desktop**: Electron with native file system access
- **Testing**: Vitest with jsdom environment

## Development Commands

### Primary Development

```bash
# Install dependencies
pnpm install

# Web development
pnpm dev                # Start web app development server
pnpm build              # Build web app for production

# Desktop development
pnpm dev:desktop        # Start desktop app (requires libsimple download first)
pnpm build:desktop      # Build desktop app
pnpm pkg:desktop        # Package desktop app for distribution

# First-time desktop setup
cd apps/desktop && node scripts/download-libsimple.cjs
```

### Code Quality

```bash
pnpm lint               # Run oxlint on entire codebase
pnpm lint:fix           # Auto-fix linting issues
pnpm typecheck          # TypeScript type checking
pnpm format:write       # Format code with Prettier
pnpm format:check       # Check code formatting
```

### Testing

```bash
pnpm test               # Run Vitest tests
```

### Version Management

```bash
pnpm patch              # Increment patch version
pnpm minor              # Increment minor version
pnpm major              # Increment major version
```

## Key Development Concepts

### Extension System

Eidos has a powerful extension architecture with three main types:

- **Blocks**: UI components for custom data display and interaction
- **Scripts**: TypeScript/JavaScript/Python processing logic that can be called by AI

Extensions are stored in the database and can be created manually or generated by AI.

### Data Architecture

- SQLite database with custom meta-tables for managing schemas, views, and extensions
- Real-time data pipeline with change triggers and full-text/semantic search
- File system integration with both browser APIs and native file system
- Undo/redo system and data synchronization capabilities

### AI Integration

- Multi-provider AI support with unified interface
- Context-aware AI functions that can access and manipulate data
- AI-powered extension generation and code assistance
- Embedding-based semantic search and RAG capabilities

## Important Development Notes

### Desktop Development Setup

For desktop development, you must first download the required SQLite extensions:

```bash
cd apps/desktop && node scripts/download-libsimple.cjs
```

### Workspace Dependencies

The project uses workspace references (e.g., `"@eidos.space/core": "workspace:*"`). Make sure to run `pnpm install` at the root level to properly link all packages.

### Code Style Guidelines

- Use TypeScript for all new code, avoid `any` types
- Follow Cursor rules in `.cursor/rules/` for coding standards
- Use Tailwind CSS for styling with mobile-first approach
- Keep components small and focused
- Use Zustand for global state, React hooks for local state
- Write tests for critical functionality using Vitest

### Testing Configuration

Tests use Vitest with jsdom environment by default, switching to node environment for specific packages like `lib/v3/*.test.ts`.

### Build Considerations

- Web builds use increased memory allocation: `NODE_OPTIONS=--max-old-space-size=8192`
- Desktop builds require specific Electron configuration and native dependencies
- Extensions are compiled and sandboxed using custom V3 compiler

Run `pnpm typecheck` and `pnpm lint` before committing to ensure code quality.
